  <action>
Create `lib/lattice_stripe/form_encoder.ex` implementing the recursive form encoder per D-11 and Pattern 4 from research. The module:

1. Public function `encode/1` accepts a map or keyword list and returns a URL-encoded string.
2. Private `flatten/2` recursively walks the data structure:
   - Maps and keyword lists: recurse with bracket-notation keys (`parent[child]`)
   - Lists of maps: index with `parent[0][key]`, `parent[1][key]`
   - Lists of scalars: index with `parent[0]`, `parent[1]`
   - Nil values: skip entirely (do not include in output)
   - All other values: convert to string with `to_string/1`
3. After flattening, map each `{key, value}` pair to `URI.encode_www_form(key)=URI.encode_www_form(to_string(value))`.
4. Sort pairs alphabetically for deterministic output (important for test assertions and caching).
5. Join with `&`.

Handle edge cases per Pitfall 2:
- Atom keys/values: convert via `to_string/1`
- Boolean: `to_string(true)` => `"true"`
- Integer/float: `to_string/1`
- Empty string: preserve (Stripe uses empty string to clear values)
- Nil: omit entirely
- Empty map/list: return `""`

Create `test/lattice_stripe/form_encoder_test.exs` with all tests from the behavior block. Use `async: true`.
