```markdown
# Error Handling
```

Sections:
1. **Brief intro**: How LatticeStripe surfaces errors. All functions return `{:ok, result} | {:error, %LatticeStripe.Error{}}`. Link to [Stripe error codes](https://docs.stripe.com/error-codes).

2. **## The Error Struct**: Show the full Error struct with all fields:
- type (atom): :card_error, :invalid_request_error, :authentication_error, :rate_limit_error, :api_error, :idempotency_error
- code (string): Stripe error code
- message (string): Human-readable message
- param (string): Parameter that caused the error
- http_status (integer): HTTP status code
- request_id (string): For Stripe support
- decline_code (string): Card decline reason
- charge (string): Charge ID for card errors
- doc_url (string): Link to Stripe docs for this error
- raw_body (string): Full response body

3. **## Pattern Matching on Error Types**: Show comprehensive case statement matching each error type with appropriate handling:
```elixir
case LatticeStripe.PaymentIntent.create(client, params) do
  {:ok, intent} ->
    intent

  {:error, %LatticeStripe.Error{type: :card_error, decline_code: code}} ->
    # Card was declined — show user-friendly message
    handle_decline(code)

  {:error, %LatticeStripe.Error{type: :authentication_error}} ->
    # API key is wrong — log and alert ops
    Logger.error("Stripe authentication failed")

  {:error, %LatticeStripe.Error{type: :rate_limit_error}} ->
    # Too many requests — back off
    Process.sleep(1000)
    retry()

  {:error, %LatticeStripe.Error{type: :invalid_request_error, param: param}} ->
    # Bad request parameters
    Logger.warning("Invalid param: #{param}")

  {:error, %LatticeStripe.Error{type: :idempotency_error}} ->
    # Idempotency key reused with different params
    Logger.error("Idempotency conflict")

  {:error, %LatticeStripe.Error{type: :api_error} = err} ->
    # Stripe server error — already retried automatically
    Logger.error("Stripe API error: #{err.message}")
end
```

4. **## Bang Variants**: Show `create!/3` raising on error, useful in scripts or when you want to crash on failure.

5. **## Automatic Retries**: Explain retry behavior:
- Default 2 retries (3 total attempts)
- Exponential backoff with jitter
- Respects Stripe-Should-Retry header
- Network errors and 5xx retried automatically
- 429 rate limit errors retried
- 409 idempotency conflicts NOT retried
- Idempotency keys reused across retries

6. **## Configuring Retries**: Show max_retries on Client.new! and per-request.

7. **## Using request_id for Support**: Show extracting request_id from error for Stripe support tickets.

8. **## Common Pitfalls**:
- Don't catch all errors — pattern match on type for appropriate handling
- :card_error is user-facing, :api_error is infrastructure — handle differently
- request_id is essential for Stripe support — always log it
- Idempotency conflicts mean you sent different params with the same key

# Telemetry

3. **## Request Events**: Document all request lifecycle events with their measurements and metadata:

**[:lattice_stripe, :request, :start]**
- Measurements: `%{system_time: integer}`
- Metadata: `%{method: atom, path: String.t, resource: String.t, operation: String.t}`

**[:lattice_stripe, :request, :stop]**
- Measurements: `%{duration: integer}` (native time units)
- Metadata: `%{method: atom, path: String.t, http_status: integer, request_id: String.t, resource: String.t, operation: String.t}`

**[:lattice_stripe, :request, :exception]**
- Measurements: `%{duration: integer}`
- Metadata: `%{kind: atom, reason: term, stacktrace: list}`

**[:lattice_stripe, :request, :retry]**
- Measurements: `%{delay: integer, attempt: integer}`
- Metadata: `%{method: atom, path: String.t, reason: term}`

4. **## Webhook Events**: Document webhook verification events:
**[:lattice_stripe, :webhook, :verify, :start]**, **:stop**, **:exception**

5. **## Custom Telemetry Handler**: Show attaching a custom handler for metrics:
```elixir
:telemetry.attach(
  "stripe-metrics",
  [:lattice_stripe, :request, :stop],
  fn _event, measurements, metadata, _config ->
    duration_ms = System.convert_time_unit(measurements.duration, :native, :millisecond)
    MyMetrics.histogram("stripe.request.duration_ms", duration_ms,
      tags: %{resource: metadata.resource, operation: metadata.operation, status: metadata.http_status}
    )
  end,
  nil
)
```

6. **## Disabling Telemetry**: Show `telemetry_enabled: false` on Client.new!. Note: webhook telemetry always fires regardless.

7. **## Common Pitfalls**:
- Duration is in native time units — use System.convert_time_unit/3
- attach_default_logger is idempotent — safe to call multiple times
- Webhook telemetry fires even with telemetry_enabled: false (infrastructure-level)
- Don't do slow work in telemetry handlers — it blocks the request

```elixir
defmodule MyApp.ReqTransport do
  @behaviour LatticeStripe.Transport

  @impl true
  def request(%{method: method, url: url, headers: headers, body: body, opts: opts}) do
    case Req.request(method: method, url: url, headers: headers, body: body) do
      {:ok, %Req.Response{status: status, headers: resp_headers, body: body}} ->
        {:ok, %{status: status, headers: resp_headers, body: body}}
      {:error, exception} ->
        {:error, exception}
    end
  end
end

# Use it
client = LatticeStripe.Client.new!(
  api_key: "sk_test_...",
  finch: MyApp.Finch,  # still needed for default, but won't be used

)
```

  @impl true

    if circuit_open?() do

    else
      if attempt < 3 do

        {true, delay}
      else
