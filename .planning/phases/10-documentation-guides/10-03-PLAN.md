```elixir
{:ok, intent} = LatticeStripe.PaymentIntent.create(client, %{

  "currency" => "usd"
})

IO.puts("PaymentIntent #{intent.id} created for $#{intent.amount / 100}")
```

6. **## Handling Errors**: Brief pattern matching example:
```elixir
case LatticeStripe.PaymentIntent.create(client, params) do
  {:ok, intent} -> # success
  {:error, %LatticeStripe.Error{type: :card_error} = err} ->
    IO.puts("Card declined: #{err.message}")
  {:error, %LatticeStripe.Error{} = err} ->
    IO.puts("API error: #{err.message}")
end
```

7. **## Next Steps**: Link to Client Configuration, Payments, Webhooks guides.

8. **## Common Pitfalls**: 
- Finch must be started before making API calls
- Amount is in cents (2000 = $20.00)
- Use test mode keys (sk_test_...) in development

**guides/client-configuration.md** (~200-300 lines)

```markdown
# Client Configuration
```

Sections:
1. **Brief intro**: All configuration via Client.new!, no global state.

2. **## Required Options**: api_key and finch. Show example.

3. **## Optional Settings**: Walk through each option:
   - `base_url` — default "https://api.stripe.com", useful for stripe-mock
   - `api_version` — override pinned version
   - `max_retries` — default 2 (3 total attempts)
   - `retry_strategy` — custom module implementing RetryStrategy behaviour
   - `timeout` — request timeout in ms
   - `telemetry_enabled` — default true
   - `stripe_account` — Connect account ID for platform operations

4. **## Per-Request Overrides**: Show opts keyword list:
```elixir
LatticeStripe.Customer.create(client, params,
  idempotency_key: "unique-key-123",
  stripe_account: "acct_connected",
  timeout: 30_000,
  expand: ["default_source"]
)
```

5. **## Multiple Clients**: Show creating different clients for different Stripe accounts/environments.

6. **## Stripe Connect**: Using stripe_account on client or per-request for platform operations. Link to Stripe Connect docs.

7. **## Common Pitfalls**:
- Client is a struct, not a process — create it and pass it around
- api_key is validated at Client.new! time — typos caught early
- Per-request opts override client defaults, not merge

```markdown
# Webhooks
```

Sections:
1. **Brief intro**: Stripe sends webhook events for async updates. Link to [Stripe Webhooks guide](https://docs.stripe.com/webhooks).

2. **## Signature Verification**: Manual verification without Plug:
```elixir
case LatticeStripe.Webhook.construct_event(raw_body, signature_header, "whsec_test_...") do
  {:ok, %LatticeStripe.Event{} = event} ->
    handle_event(event)
  {:error, %LatticeStripe.Webhook.SignatureVerificationError{} = err} ->
    # Invalid signature — reject
end
```

3. **## Using the Webhook Plug**: Show the recommended approach with LatticeStripe.Webhook.Plug:
```elixir
# In your Phoenix router or endpoint
plug LatticeStripe.Webhook.Plug,
  path: "/webhooks/stripe",
  secret: "whsec_test_...",
  handler: MyApp.StripeWebhookHandler
```

4. **## Implementing a Handler**: Show implementing the Handler behaviour:
```elixir
defmodule MyApp.StripeWebhookHandler do
  @behaviour LatticeStripe.Webhook.Handler

  @impl true
  def handle_event(%LatticeStripe.Event{type: "payment_intent.succeeded"} = event) do
    # Handle successful payment
    :ok
  end

  def handle_event(%LatticeStripe.Event{type: "checkout.session.completed"} = event) do
    # Handle checkout completion
    :ok
  end

  def handle_event(_event), do: :ok
end
```

5. **## Raw Body Caching**: Explain the Plug.Parsers raw body problem. Show CacheBodyReader setup:
```elixir
plug Plug.Parsers,
  parsers: [:json],
  pass: ["application/json"],
  body_reader: {LatticeStripe.Webhook.CacheBodyReader, :read_body, []},
  json_decoder: Jason
```

6. **## Dynamic Secrets**: Show using secret_mfa for runtime secret resolution:
```elixir
plug LatticeStripe.Webhook.Plug,
  path: "/webhooks/stripe",
  secret_mfa: {MyApp.Config, :stripe_webhook_secret, []},
  handler: MyApp.StripeWebhookHandler
```

7. **## Phoenix Integration Example**: Complete Phoenix endpoint setup showing CacheBodyReader + Plug.Parsers + router with Webhook.Plug.

8. **## Tolerance Window**: Show configuring tolerance (default 300s):
```elixir
LatticeStripe.Webhook.construct_event(payload, sig, secret, tolerance: 600)
```

9. **## Common Pitfalls**:
- Raw body must be preserved for signature verification (Plug.Parsers consumes it)
- CacheBodyReader must be configured BEFORE Plug.Parsers
- Webhook secrets start with whsec_ — don't confuse with API keys
- Return 200 quickly from webhook handler — do heavy processing async
- Test with `LatticeStripe.Testing.generate_webhook_payload/3` in your test suite
