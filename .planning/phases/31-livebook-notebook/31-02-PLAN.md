**6. Webhooks section (`## Webhooks`)**

Prose intro: "Webhooks let you react to events in Stripe (payment succeeded, subscription cancelled, etc.). LatticeStripe provides `Webhook.construct_event/4` for HMAC signature verification — use it in your Phoenix endpoint."

Prose: "This section demonstrates the pure-function verification API. No live webhook server is needed — we use hardcoded test inputs."

```elixir
# Webhook signature verification is a pure function — no HTTP needed
raw_body = ~s({"id":"evt_test_123","object":"event","type":"payment_intent.succeeded","data":{"object":{"id":"pi_test"}}})

# In production, these come from the request:
# - raw_body: the raw HTTP request body (not parsed JSON)
# - sig_header: the Stripe-Signature header value
# - secret: your webhook endpoint signing secret from the Stripe dashboard

# With an invalid signature, construct_event returns an error:
case LatticeStripe.Webhook.construct_event(raw_body, "t=1234,v1=invalid", "whsec_test_secret") do
  {:ok, %LatticeStripe.Event{} = event} ->
    IO.puts("Event type: #{event.type}")
  {:error, reason} ->
    IO.puts("Verification failed: #{inspect(reason)}")
end
```

Prose after: "In a Phoenix app, use the `LatticeStripe.WebhookPlug` in your endpoint pipeline. See `guides/webhooks.md` for the full Plug setup."

**7. v1.2 Highlights section (`## v1.2 Highlights`)**

Prose intro: "LatticeStripe v1.2 adds several developer experience improvements. Here are the highlights."

Sub-section `### Expand Deserialization` — per D-08:

Prose: "When you pass `expand:` options, v1.2 returns fully typed structs instead of string IDs."

```elixir
# Without expand — customer field is a string ID
{:ok, sub_resp} = LatticeStripe.Subscription.list(client, %{"customer" => customer.id, "limit" => "1"})
sub_for_expand = List.first(sub_resp.data.data)
IO.puts("Customer field (no expand): #{inspect(sub_for_expand.customer)}")
# => "cus_..." (string ID)

# With expand — customer field becomes a full %Customer{} struct
{:ok, expanded_resp} = LatticeStripe.Subscription.retrieve(
  client,
  sub_for_expand.id,
  expand: ["customer"]
)
expanded = expanded_resp.data
IO.puts("Customer field (expanded): #{inspect(expanded.customer.__struct__)}")
Kino.Tree.new(expanded)
```

Sub-section `### Concurrent Requests with Batch` — per D-08:

Prose: "Batch.run/3 executes multiple API calls concurrently using Task.async_stream. Each call is isolated — one failure doesn't crash the others."

```elixir
{:ok, results} =
  LatticeStripe.Batch.run(client, [
    {LatticeStripe.Customer, :retrieve, [customer.id]},
    {LatticeStripe.Subscription, :list, [%{"customer" => customer.id}]},
    {LatticeStripe.Invoice, :list, [%{"customer" => customer.id}]}
  ])

for {label, result} <- Enum.zip(["Customer", "Subscriptions", "Invoices"], results) do
  case result do
    {:ok, _data} -> IO.puts("#{label}: success")
    {:error, err} -> IO.puts("#{label}: error - #{err.message}")
  end
end
```
