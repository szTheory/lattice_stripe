**classify_operation/1 private function (lib/lattice_stripe/client.ex):**
Add after the last private function in the module (near end of file, after `parse_stripe_should_retry/1` or equivalent). This mirrors the pattern from Telemetry's `parse_resource_and_operation/2` but returns atoms per D-02:

```elixir
defp classify_operation(%Request{method: method, path: path}) do
  segments =
    path
    |> String.replace_prefix("/v1/", "")
    |> String.replace_prefix("/v1", "")
    |> String.split("/", trim: true)

  case {method, segments} do
    {:get, [_resource]}           -> :list
    {:get, [_resource, "search"]} -> :search
    {:get, [_resource, _id]}      -> :retrieve
    {:post, [_resource]}          -> :create
    {:post, [_resource, _id]}     -> :update
    {:delete, [_resource, _id]}   -> :delete
    _                             -> :other
  end
end
```

**Timeout resolution update (lib/lattice_stripe/client.ex, line 177):**
Replace `effective_timeout = Keyword.get(req.opts, :timeout, client.timeout)` with:

```elixir
effective_timeout =
  case Keyword.fetch(req.opts, :timeout) do
    {:ok, t} ->
      t

    :error ->
      case client.operation_timeouts do
        %{} = timeouts ->
          op_type = classify_operation(req)
          Map.get(timeouts, op_type, client.timeout)

        nil ->
          client.timeout
      end
  end
```

IMPORTANT per Pitfall 3: `classify_operation/1` is only called inside the `%{} = timeouts` branch, NOT when `operation_timeouts` is `nil`. The nil path has zero extra work — critical for the hot path.
