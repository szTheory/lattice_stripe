Per D-05, D-06, D-07: Create `lib/lattice_stripe/testing.ex` as a PUBLIC module (in lib/, not test/support/) that ships with the hex package.

The module provides two functions:

**`generate_webhook_event(type, object_data \\ %{}, opts \\ [])`**
- Accepts event type string (e.g., `"payment_intent.succeeded"`)
- Accepts object_data map for the `data.object` field
- Options: `:id` (default: `"evt_test_" <> random_hex(16)`), `:api_version` (default: `"2026-03-25.dahlia"`), `:livemode` (default: `false`)
- Builds a raw map with keys: `"id"`, `"object"` ("event"), `"type"`, `"api_version"`, `"created"` (System.system_time(:second)), `"livemode"`, `"pending_webhooks"` (1), `"request"` (%{"id" => nil, "idempotency_key" => nil}), `"data"` (%{"object" => object_data})
- Calls `Event.from_map/1` on the raw map
- Returns `%Event{}` struct

**`generate_webhook_payload(type, object_data \\ %{}, opts)`**
- `:secret` option is REQUIRED (use `Keyword.pop!`)
- Optional `:timestamp` (default: `System.system_time(:second)`)
- Remaining opts forwarded to `generate_webhook_event/3`
- Builds the raw map BEFORE calling Event.from_map/1 (per RESEARCH.md Open Question 3 -- encode the raw map, not the struct, to avoid round-trip encoding issues)
- Calls `Jason.encode!/1` on the raw map to get `payload` string
- Calls `Webhook.generate_test_signature(payload, secret, timestamp: timestamp)` to get `sig_header`
- Returns `{payload, sig_header}` tuple

**Private helper:** `defp random_hex(bytes), do: :crypto.strong_rand_bytes(bytes) |> Base.encode16(case: :lower)`

**Module attributes:**
- `@moduledoc` with usage examples showing both `generate_webhook_event/2` and `generate_webhook_payload/3` patterns
- `@doc` on both public functions with parameter descriptions, return types, and examples
- `@spec` on both public functions
- `alias LatticeStripe.{Event, Webhook}`

**Test file:** `test/lattice_stripe/testing_test.exs`
- `use ExUnit.Case, async: true`
- Test all 8 behaviors listed above
- For round-trip test (Test 7): call `generate_webhook_payload/3` with a known secret, then call `Webhook.construct_event(payload, sig_header, secret)` and assert `{:ok, %Event{}}`
