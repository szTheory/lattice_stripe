Purpose: Unblock Elixir developers building subscription-driven SaaS by delivering a pattern-matchable, PII-safe, idempotent, proration-guarded Subscription API that matches the Phase 14 Invoice precedent.

Output:

- Three new typed nested struct modules (`PauseCollection`, `CancellationDetails`, `TrialSettings`) each with unit tests
- Extended `Billing.Guards.has_proration_behavior?/1` handling `items[]` arrays plus new guard tests

- Resolved open questions OQ-1/OQ-2/OQ-3 in the decisions section below
</objective>

<decisions>
## Planner Decisions (resolving RESEARCH open questions)

- **OQ-1 — RESOLVED: YES.** Ship both `Subscription.cancel/3` (2-arity-after-client convenience) and `Subscription.cancel/4`. The 3-arity delegates to `cancel/4` with `%{}` params. Matches `Checkout.Session.expire/3` precedent.
- **OQ-2 — RESOLVED: YES, require `subscription` param** via `Resource.require_param!/3` on `SubscriptionItem.list/3` (implemented in Plan 15-02). Unfiltered subscription-item listing is an antipattern and stripity_stripe users have complained about accidental full-list queries. This plan records the decision; Plan 15-02 implements it.
- **OQ-3 — RESOLVED: NO new telemetry events.** Subscription CRUD piggybacks on the general `[:lattice_stripe, :request, :start|:stop|:exception]` events emitted by `Client.request/2`. Subscription state transitions belong to user webhook handlers, not the SDK layer. This is explicitly documented in `guides/subscriptions.md` (Plan 15-03).
</decisions>
