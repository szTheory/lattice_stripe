**Module structure:**
```elixir
defmodule LatticeStripe.Batch do
  @moduledoc """
  Execute multiple Stripe API calls concurrently.
  ...
  """

  alias LatticeStripe.{Client, Error}

  @type task :: {module(), atom(), [term()]}
  @type result :: {:ok, term()} | {:error, Error.t()}

  @spec run(Client.t(), [task()], keyword()) :: {:ok, [result()]} | {:error, Error.t()}
  def run(%Client{} = client, tasks, opts \\ []) when is_list(tasks) do
    with :ok <- validate_tasks(tasks) do
      max_concurrency = Keyword.get(opts, :max_concurrency, System.schedulers_online())

      results =
        tasks
        |> Task.async_stream(
          fn {mod, fun, args} ->
            try do
              apply(mod, fun, [client | args])
            rescue
              e ->
                {:error,
                 %Error{
                   type: :connection_error,
                   message: "Task raised exception: #{Exception.message(e)}"
                 }}
            end
          end,
          max_concurrency: max_concurrency,
          ordered: true,
          timeout: :infinity,
          on_timeout: :kill_task
        )
        |> Enum.map(&map_stream_result/1)

      {:ok, results}
    end
  end
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller -> Batch.run/3 | Untrusted MFA tuples passed by application code |
| Batch -> apply(mod, fun, args) | Dynamic dispatch via `apply/3` on caller-provided atoms |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-01 | Tampering | `apply(mod, fun, [client \| args])` | accept | MFA tuples come from the application's own code (same trust level as calling resource functions directly). `valid_mfa?/1` guards ensure structural validity (atom module, atom function, list args). No string-to-atom conversion — no atom table exhaustion. The caller could already call `apply/3` directly; Batch adds no new attack surface. |
| T-27-02 | Denial of Service | `max_concurrency` option | accept | Default is `System.schedulers_online()` (bounded). Caller-provided values are the caller's responsibility — they control their own concurrency budget. No unbounded spawning possible since `Task.async_stream` enforces the cap. |
| T-27-03 | Elevation of Privilege | Task processes | mitigate | Each task runs in a linked-then-monitored process via `Task.async_stream`. `on_timeout: :kill_task` ensures timed-out tasks are terminated. `try/rescue` inside the task body prevents exception propagation. Caller process is never crashed by task failures. |
</threat_model>
