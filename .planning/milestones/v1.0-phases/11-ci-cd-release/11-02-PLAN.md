3. Create `.github/workflows/release.yml` (per D-13 through D-21, RESEARCH Pattern 2):

```yaml
name: Release

on:
  push:
    branches: [main]

permissions:
  contents: write
  pull-requests: write

jobs:
  release-please:
    name: Release Please
    runs-on: ubuntu-latest
    outputs:
      release_created: ${{ steps.release.outputs.release_created }}
      tag_name: ${{ steps.release.outputs.tag_name }}
    steps:
      - uses: googleapis/release-please-action@v4
        id: release
        with:
          command: manifest
          token: ${{ secrets.GITHUB_TOKEN }}

  publish-hex:
    name: Publish to Hex.pm
    runs-on: ubuntu-latest
    needs: release-please
    if: ${{ needs.release-please.outputs.release_created == 'true' }}
    steps:
      - uses: actions/checkout@v4
      - uses: erlef/setup-beam@v1
        with:
          elixir-version: '1.19'
          otp-version: '28'
      - run: mix deps.get
      - run: mix hex.publish --yes
        env:
          HEX_API_KEY: ${{ secrets.HEX_API_KEY }}
```

Key details:
- Elevated permissions (`contents: write`, `pull-requests: write`) only in this workflow, not ci.yml (per D-08)
- `command: manifest` reads both config files (per D-14)
- `publish-hex` job gates on `release_created == 'true'` output from release-please job
- `mix hex.publish --yes` avoids interactive prompt (per RESEARCH anti-pattern)
- HEX_API_KEY as repo secret (per D-15)
- `mix hex.publish` publishes docs to HexDocs automatically (per D-21)

2. Create `.github/workflows/dependabot-automerge.yml` (per D-25, D-35, RESEARCH Pattern 5):

```yaml
name: Dependabot Auto-Merge

on: pull_request

permissions:
  contents: write
  pull-requests: write

jobs:
  auto-merge:
    runs-on: ubuntu-latest
    if: github.actor == 'dependabot[bot]'
    steps:
      - name: Dependabot metadata
        id: metadata
        uses: dependabot/fetch-metadata@v2
        with:
          github-token: "${{ secrets.GITHUB_TOKEN }}"
      - name: Enable auto-merge for patch updates
        if: steps.metadata.outputs.update-type == 'version-update:semver-patch'
        run: gh pr merge --auto --squash "$PR_URL"
        env:
          PR_URL: ${{ github.event.pull_request.html_url }}
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Key details:
- Only triggers for `dependabot[bot]` actor
- Uses `dependabot/fetch-metadata@v2` to check update type
- Only auto-merges `semver-patch` updates (per D-25)
- Uses `--squash` to match repo merge strategy (per D-35)
- `--auto` gates on required CI checks passing before merge
